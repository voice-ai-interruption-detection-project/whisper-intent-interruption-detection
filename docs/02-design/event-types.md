# Event Types & Intents: 사용자 신호 분류 체계

## Event Type: 고객이 보낸 신호의 종류

고객이 AI 발화 중에 "어떤 종류의 신호를 보냈는가"를 7가지로 분류.

---

## 7가지 Event Type

### 1️⃣ `no_speech` — 사용자 발화 없음

**정의:** 고객이 말을 하지 않음

**발생 조건:**
- AI가 말하는 동안 고객이 침묵
- 음성 신호 감지 없음

**AI의 자연스러운 행동:** `continue`

**예시:**
```
AI: "배송은 내일 도착 예정입니다."
고객: (침묵)
```

---

### 2️⃣ `noise` — 배경음 또는 비언어 소리

**정의:** 고객이 의도적으로 말하지 않았지만 음성 신호가 감지됨

**발생 조건:**
- 기침, 숨소리
- 주변 배경음 (문 닫는 소리, 다른 사람 목소리)
- 짧은 비명 ("어", "으")

**AI의 자연스러운 행동:** `continue`

**예시:**
```
AI: "상품은 어떤..."
고객: *기침*
```

---

### 3️⃣ `backchannel` — 맞장구

**정의:** 고객이 동의/인정하며 짧은 반응

**발생 조건:**
- "네", "음", "네요", "맞아", "알겠어요"
- "그래", "그렇군요"
- 고개 끄덕이는 소리

**고객의 의도:** "계속해, 나는 동의해"

**AI의 자연스러운 행동:** `continue` / `brief_ack`

**예시:**
```
AI: "배송 상황을 알려드리겠습니다."
고객: "네."
```

---

### 4️⃣ `same_intent_question` — 같은 주제 내 보충 질문

**정의:** 고객이 AI의 현재 의도와 같은 주제 내에서 추가 질문

**발생 조건:**
- AI 주제: "배송조회" → 고객 질문: "배송비는?"
- AI 주제: "상품정보" → 고객 질문: "색상은 뭐가 있어?"

**고객의 의도:** "같은 주제인데 내 질문에 답해"

**AI의 자연스러운 행동:** `pause` (멈추고 답하기)

**예시:**
```
AI: "현재 배송 중이며 내일 도착 예정입니다."
고객: "배송비는 따로 드나요?"
```

---

### 5️⃣ `intent_shift` — 다른 의도로의 전환

**정의:** 고객이 AI의 현재 의도와 다른 새로운 의도를 제시

**발생 조건:**
- AI 의도: "배송조회" → 고객 의도: "환불요청"
- AI 의도: "상품정보" → 고객 의도: "배송조회"

**고객의 의도:** "내가 원한 건 이게 아니야, 다른 것 해줘"

**AI의 자연스러운 행동:** `stop_and_switch` (주제 전환)

**예시:**
```
AI: "현재 배송 중입니다."
고객: "아, 그게 아니라 환불받고 싶은데요."
```

---

### 6️⃣ `complaint` — 불만 또는 긴급 발화

**정의:** 고객이 불평, 불만, 또는 긴급한 상태를 표현

**발생 조건:**
- "뭐 이딴 배송이야!"
- "지금 당장 해줘야 돼!"
- "이건 말이 안 된다"

**고객의 의도:** 감정 상태 표현, 우선 조치 필요

**AI의 자연스러운 행동:** `stop_and_switch` 또는 `handoff` (심각하면 상담사 연결)

**예시:**
```
AI: "배송은 정상 진행 중입니다."
고객: "뭐 이딴 배송이야! 언제 오는 거야!?"
```

---

### 7️⃣ `ambiguous` — 의도가 불명확한 발화

**정의:** 고객의 발화 의도를 명확히 파악하기 어려움

**발생 조건:**
- "음... 잠깐요."
- "아, 그게..."
- 한두 글자만 말함

**고객의 의도:** 명확하지 않음

**AI의 자연스러운 행동:** `ask_clarifying` (확인 질문)

**예시:**
```
AI: "어떤 상품을 찾으세요?"
고객: "음... 잠깐요."
```

## Intent: 고객의 비즈니스 의도

Event Type과 함께 중요한 것은 **"고객이 어떤 비즈니스 의도를 가지고 있는가"**.

### 6가지 Intent (커머스 도메인)

| Intent | 정의 | 예시 발화 |
|--------|------|---------|
| `shipping_inquiry` | 배송 상태 조회 | "언제 도착해요?", "배송 어디까지 왔어?" |
| `refund_request` | 환불 요청 | "환불받고 싶어요", "돈 돌려줄 수 있나?" |
| `return_request` | 반품 요청 | "반품하고 싶어요", "수거는?" |
| `payment_issue` | 결제 문제 | "결제가 두 번 됐어요", "카드 취소됐나?" |
| `product_inquiry` | 상품 문의 | "사이즈가 맞나요?", "색상 뭐가 있어?" |
| `agent_connection` | 상담사 연결 | "사람이랑 통화하고 싶어요" |

## Intent Shift Detection: 의도 변화 감지

**핵심:** Policy v2 정책에서 사용되는 개념

### 의도 유사도 계산

SBERT (Sentence-BERT)를 사용해 의도 간 유사도를 계산:

```
AI 현재 의도: "배송조회"
고객 텍스트: "환불받고 싶은데요"
→ 예측 의도: "환불요청"

유사도 = similarity("배송조회", "환불요청")
       = 0.22  (매우 낮음)

판단: intent_shift (의도 전환) → stop_and_switch
```

### Threshold 설정

```
유사도 >= 0.6: 같은 의도 (same_intent_question)
유사도 < 0.6:  다른 의도 (intent_shift)
```

## Event Type과 Intent의 관계

```
Event Type: "어떤 종류의 신호가 들어왔는가?"
  ↓
Intent: "그 신호 안에 어떤 의도가 담겨있는가?"
  ↓
Action: "그러면 AI는 뭘 해야 하는가?"
```

**예시:**
```
고객 발화: "아, 그게 아니라 환불받고 싶은데요."

1️⃣ Event Type 분류
   - 음성 신호 있음 ✓
   - "환불받고 싶어요" → intention 명확 ✓
   - AI 의도와 다른가? → 유사도 0.22 (낮음) ✓
   → event_type = "intent_shift"

2️⃣ Intent 추출
   → predicted_intent = "환불요청"

3️⃣ Action 결정
   → expected_action = "stop_and_switch"
```
