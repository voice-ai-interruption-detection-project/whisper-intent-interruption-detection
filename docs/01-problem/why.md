# 왜 이 문제인가?

## 배경: 음성 기반 상담 시스템의 확대

### 현실
- **Kakao i, Naver ClovaBot** 등 이미 상용화된 Voice Agent 존재
- **AWS Connect** 같은 클라우드 기반 솔루션도 음성 상담 지원
- 금융권에서는 이미 간단한 문의의 경우 AI 상담원으로 대체하도록 처리 중

---

## 현실의 문제: 어색한 대화

### 상황 1: 맞장구인데 멈춤
```
AI: "현재 배송 중이며 내일 오후 도착 예정입니다."
고객: "네, 알겠어요."  (동의)
AI: 🤐 (갑자기 멈춤)
고객: "???"
```

### 상황 2: 요청을 못 들음
```
AI: "배송 현황을 알려드리겠습니다."
고객: "아, 그게 아니라 환불받고 싶은데요."  (다른 요청)
AI: "현재 상품은 배송 중이고..."  (계속 배송 얘기)
고객: 😠
```

---

## 근본 원인: 사용자의 의도를 보지 않는다

### 현재 대부분의 시스템
1. 음성 감지 → 멈춤 (간단한 방식)
2. 또는 STT는 함 → 하지만 의도 판단 안 함

### 문제
- "네, 알겠어요" vs "환불받고 싶어요" 구분 불가
- 고객이 **뭐라고 말했는지**는 아는데, **의도가 뭔지**는 못 봄

### 따라서 필요한 것
```
음성이 있다
  ↓
뭐라고 말했는가 (텍스트)
  ↓
의도가 뭔가 (의도 분류)
  ↓
상황에 맞는 행동 (동의면 계속, 다른 요청이면 전환)
```
