# VAD-only Baseline의 한계

## 현재 표준

**"음성이 있으면 멈춘다"** — 간단하지만 고객의 의도를 모른다.

---

## 한계 1: 맞장구를 멈춤으로 인식 (False Stop)

```
AI: "상품은 내일 도착 예정입니다."
고객: "네, 알겠어요."  (맞장구)
```

AI는 음성을 감지하면 무조건 멈추기 때문에, 고객의 동의 신호를 '개입'으로 착각한다.

**결과:** 부자연스러운 대화, False Stop Rate ~25%

---

## 한계 2: 의도 전환을 놓침 (Missed Switch)

```
AI: "현재 배송 중입니다."
고객: "아, 그게 아니라 환불받고 싶은데요."  (의도 전환)
```

AI는 "뭔가 소리가 났다"는 것만 알 뿐, **무엇을 말했는지 모른다.** 그래서 계속 배송 얘기를 한다.

**결과:** 고객 만족도 급락, Missed Switch Rate ~45%

---

## 한계 3: 상황별 대응 불가

같은 음성 신호라도 상황에 따라 AI의 대응이 달라야 한다:

- **같은 주제 내 질문**: "배송비는 따로 드나요?" → 잠깐 멈추고 답하기
- **다른 주제로 전환**: "환불받고 싶어요" → 주제를 바꾸고 새로 시작
- **불만/긴급**: "뭐 이딴 배송이야!" → 상담사 연결

하지만 VAD-only 방식은 **모두 '음성이 났다'고만 본다.**

---

## 무엇이 필요한가?

- 고객이 **무엇을 말했는가** (텍스트)
- 고객의 **의도가 무엇인가** (의도 분류)
- 이전 주제와의 **관계는 무엇인가** (맥락)
