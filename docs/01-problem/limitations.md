# 단순 음성 감지 방식의 한계

## 문제는 "소리가 났는가"만이 아니다

음성 AI 상담에서 고객이 말하는 소리를 감지하는 것만으로는 충분하지 않습니다.

같은 음성 신호라도 의미가 다르면 AI 행동도 달라져야 합니다.

| 고객 신호 | 자연스러운 AI 행동 |
| --- | --- |
| 침묵 또는 배경음 | 하던 말을 계속한다 |
| "네", "음" 같은 맞장구 | 계속하거나 짧게 인정하고 이어간다 |
| 같은 업무 안의 질문 | 질문에 답하고 원래 흐름으로 돌아간다 |
| 다른 업무 요청 | 현재 발화를 멈추고 새 흐름으로 전환한다 |
| 불만 또는 긴급 발화 | 우선 대응하거나 상담사 연결 후보로 보낸다 |
| 모호한 발화 | 확인 질문을 한다 |

## 한계 1: 맞장구를 과한 개입으로 볼 수 있다

```text
AI: 상품은 내일 도착 예정입니다.
고객: 네, 알겠어요.
```

이 발화는 새 요청이 아니라 수신 확인에 가깝습니다. AI가 여기서 흐름을 크게 전환하면 대화가 끊깁니다.

## 한계 2: 의도 전환을 놓칠 수 있다

```text
AI: 현재 배송 중입니다.
고객: 아, 그게 아니라 환불받고 싶은데요.
```

고객은 배송이 아니라 환불 업무로 바꾸려 합니다. AI가 배송 안내를 계속하면 고객은 자신의 요청이 무시됐다고 느낍니다.

## 한계 3: 같은 질문과 다른 업무를 구분해야 한다

```text
AI current intent: shipping_inquiry
고객: 배송비는 얼마예요?
```

이 경우는 같은 배송 업무 안의 보충 질문이므로 `respond_and_continue`가 자연스럽습니다.

```text
AI current intent: shipping_inquiry
고객: 환불받고 싶어요.
```

이 경우는 새 업무 요청이므로 `stop_and_switch`가 자연스럽습니다.

## 필요한 구조

Wind는 음성 감지 자체보다 그 뒤의 판단 구조를 검증합니다.

```text
transcript / speech signal
-> 고객 신호 해석
-> AI 행동 선택
-> expected_actions와 actual_action 비교
-> 실패 분석
```
